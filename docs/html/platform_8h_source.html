289  /*
290  * The graphics implementation type defines how graphics acceleration will be
291  * provided to the Android container.
292  *
293  * Anbox supports different ways of exposing graphic acceleration to the Android
294  * container. Graphics acceleration can either be provided through a translation
295  * layer where all access to the GPU happens on the Anbox side and Android has
296  * no direct access to the GPU. Alternatively Android can get direct GPU access
297  * and perform all graphics acceleration natively.
298  *
299  * If not provided by a platform implementation, Anbox will default to
300  * ANBOX_GRAPHICS_IMPLEMENTATION_TYPE_HOST_RENDERING
301  *
302  * The value of this configuration item is of type `AnboxGraphicsImplementationType`
303  */
304  GRAPHICS_IMPLEMENTATION_TYPE = 9,
305 
306  /*
307  * Configuration details for native graphics acceleration

309  * The platform can further detail on how nativ graphics acceleration will be
310  * made available inside the Android container. This allows defining GL/Vulkan
311  * driver implementation names and other things.
312  *
313  * The value of this configuration item is of type `AnboxNativeGraphicsConfiguration`
314  */
